Lumi еще не выпущен, он доступен в стадии разработки.

Если вы уже используете Linux и хотите быстро запустить Lumi, используйте последнюю версию **AppImage** разработки из артефактов GitLab:

https://gitlab.gnome.org/pixelmixer/lumi-dev/-/artifacts

1. Загрузите последнюю версию архива артефактов AppImage.
2. Распакуйте молнию.
3. Дважды щелкните файл `Lumi*.AppImage`, чтобы запустить его.

AppImage уже должен быть работоспособным. Если это не так, включите **Разрешить выполнение файла как программы** в разрешениях файла или используйте метод терминала, указанный ниже.

```bash
chmod +x Lumi*.AppImage
./Lumi*.AppImage
```

## Настройка Wacom в Linux

Для цифровой живописи в Lumi обычно лучше всего подходит простая **настройка линейного давления**:

- Сохраняйте кривую давления драйвера планшета линейной.
- Сохраняйте линейные кривые давления/входа в Lumi.
- Формируйте ощущения с помощью самой кисти, поскольку динамика кисти может быть нелинейной.

Вы можете проверить и сбросить кривую драйвера Linux с помощью:

```bash
xsetwacom --get "Wacom Intuos Pro L Pen stylus" PressureCurve
xsetwacom --set "Wacom Intuos Pro L Pen stylus" PressureCurve 0 0 100 100
```

Практические советы:

- Lumi в настоящее время блокирует проблемный ввод с панели/сенсорного кольца Wacom, чтобы избежать сбоев X11. Вместо этого сопоставьте кнопки планшета с **относительным** размером кисти вверх/вниз.
- Если перетаскивание размером кисти с помощью `Alt` не работает, возможно, ваш рабочий стол использует `Alt` для перемещения окон. Измените этот ярлык оконного менеджера на `Super` или отключите его.

Если вы хотите работать с исходным кодом, перейдите по адресу [Technical Guides](/hub/technical-guides/) и [Installation](/hub/technical-guides/Installation/).
