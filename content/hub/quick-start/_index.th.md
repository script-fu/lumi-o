Lumi ยังไม่วางจำหน่าย แต่มีให้บริการเป็นเวอร์ชันพัฒนา

หากคุณใช้ Linux อยู่แล้วและต้องการเรียกใช้ Lumi อย่างรวดเร็ว ให้ใช้ **development AppImage** ล่าสุดจากอาร์ติแฟกต์ GitLab:

https://gitlab.gnome.org/pixelmixer/lumi-dev/-/artifacts

1. ดาวน์โหลด zip สิ่งประดิษฐ์ AppImage การพัฒนาล่าสุด
2. แตกซิปออก
3. ดับเบิลคลิกไฟล์ `Lumi*.AppImage` เพื่อเรียกใช้

AppImage ควรใช้งานได้แล้ว หากไม่เป็นเช่นนั้น ให้เปิดใช้งาน **อนุญาตให้เรียกใช้ไฟล์ในรูปแบบโปรแกรม** ในการอนุญาตของไฟล์ หรือใช้วิธีเทอร์มินัลด้านล่าง

```bash
chmod +x Lumi*.AppImage
./Lumi*.AppImage
```

## การตั้งค่า Wacom บน Linux

สำหรับการวาดภาพดิจิทัลใน Lumi โดยทั่วไปแล้ว **การตั้งค่าแรงดันเชิงเส้น** แบบง่ายๆ จะดีที่สุด:

- รักษาเส้นโค้งแรงดันของไดรเวอร์แท็บเล็ตให้เป็นเส้นตรง
- รักษาเส้นโค้งความดัน/อินพุตให้เป็นเส้นตรง Lumi
- สร้างความรู้สึกด้วยตัวแปรงเอง เนื่องจากไดนามิกของแปรงอาจไม่เป็นเชิงเส้น

คุณสามารถตรวจสอบและรีเซ็ตเส้นโค้งไดรเวอร์ Linux ด้วย:

```bash
xsetwacom --get "Wacom Intuos Pro L Pen stylus" PressureCurve
xsetwacom --set "Wacom Intuos Pro L Pen stylus" PressureCurve 0 0 100 100
```

เคล็ดลับการปฏิบัติ:

- ปัจจุบัน Lumi บล็อกอินพุตแพด/วงแหวนสัมผัสของ Wacom ที่เป็นปัญหา เพื่อหลีกเลี่ยงข้อผิดพลาด X11 แมปปุ่มแท็บเล็ตกับ **สัมพันธ์** ขนาดแปรงขึ้น/ลงแทน
- หากการลากขนาดแปรงด้วย `Alt` ไม่ทำงาน เดสก์ท็อปของคุณอาจใช้ `Alt` เพื่อย้ายหน้าต่าง เปลี่ยนทางลัดตัวจัดการหน้าต่างนั้นเป็น `Super` หรือปิดใช้งาน
