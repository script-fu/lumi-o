ワークスペースは、UI 環境全体の保存されたスナップショットです。これには、開いているパネルと場所、標準表示と全画面表示の両方のキャンバスの装飾とパディング、アクティブなテーマとアイコン セット、ツールボックスのレイアウト、アクティブなパレット、ツール設定が含まれます。 Lumi を使用すると、名前付きワークスペースを好きなだけ保存し、それらを即座に切り替えることができます。開いているすべてのイメージが適切に更新され、再起動する必要はありません。

## ワークスペースが節約できるもの

名前付きの各ワークスペースには、以下が個別に保存されます。

|コンポーネント |内容 |
| :--- | :--- |
| **レイアウト** |ウィンドウの位置とサイズ、ドックの配置 (左右のパネル列、開いているパネルとその順序)、シングル ウィンドウ モードとマルチ ウィンドウ モード、最大化状態、タブ バーの表示と位置 |
| **ツール オプション** |各ツールの現在の設定 (ブラシ サイズ、硬さ、ワープ動作など) |
| **入力デバイス** |入力デバイスの構成: 圧力曲線、ボタンの割り当て、スタイラスやその他のデバイスの軸マッピング |
| **キャンバス装飾** |ルーラー、スクロールバー、ガイド、グリッド、選択範囲のハイライト、レイヤー境界、およびキャンバス境界のワークスペースごとのデフォルト — **環境設定 → 画像ウィンドウ → デフォルトの外観** および **全画面表示** で、標準表示と全画面表示で個別に設定します。
| **キャンバス パディング** |通常および全画面表示のワークスペースごとのパディング モードと色 — **[設定] → [画像ウィンドウ] → [デフォルトの外観]** で設定 |
| **テーマとアイコン** |アクティブなテーマ、暗い/明るい色のバリエーション、アイコン セット、アイコン サイズのオーバーライド、フォント スケール |
| **ツールボックス** | FG/BG ウィジェットの位置 (上/下/左/右)、FG/BG スケール、Wilber マスコットの可視性、ツール グループ ヘッダー |

アクティブな **パレット** と **ツール プリセット** もワークスペースごとに記録され、切り替えると復元されます。

> **キャンバスの装飾とパディング**はによって制御されます
> **設定 → 画像ウィンドウ → 詳細ウィンドウ オプション → デフォルトの外観** (標準ビュー)
> および **全画面表示** (全画面表示)。これらの設定を好みに合わせて調整してください。
> 次に、ワークスペースを保存します。 **表示メニュー** 項目 (ルーラー、ガイドなど) は、
> 現在の画像ウィンドウはワークスペースごとに保存されません。

### スイッチのライブアップデート

ワークスペースを切り替えると、開いているすべての画像ウィンドウがすぐに更新されます。ルーラー、ガイド、スクロールバー、パディングの色、その他すべてのビュー設定は、画像を閉じて再度開く必要がなく、その場で変更されます。

## アクセス

**編集→設定→ワークスペース**

「ワークスペース設定」ページの上部セクションには、保存されているすべてのワークスペースがリストされ、それらを管理するためのコントロールが提供されます。

## ワークスペースの作成

パネル、ツール、パレットを希望どおりに設定し、次の操作を行います。

1. **編集 → 環境設定 → ワークスペース** を開きます。
2. [**名前を付けてレイアウトを保存…**] をクリックします。
3. 名前を入力し、**保存**をクリックします。

新しいワークスペースが [**アクティブ レイアウト**] ドロップダウンと [**Windows**] メニューに表示されます。

## ワークスペースの切り替え

切り替える方法は 2 つあります。

- **Windows メニュー**: レイアウト名は **Windows → Layout** の下に表示され、キャンバスから簡単にアクセスできます。
- **設定 → ワークスペース**: **アクティブなレイアウト** ドロップダウンからレイアウトを選択し、**レイアウトの再読み込み** をクリックします。

切り替えは即時に行われます — Lumi はパネル レイアウトの再構築、ツール オプションの復元、デバイス設定のリロード、キャンバスの装飾、パディング、テーマ、ツールボックス レイアウトの更新をすべて再起動せずに行います。

## ワークスペースの管理

**編集 → 環境設定 → ワークスペース** から:|アクション |効果 |
| :--- | :--- |
| **レイアウトを保存** |現在のワークスペースを現在の設定で上書きします。 |
| **レイアウトに名前を付けて保存…** |現在の設定から新しい名前付きワークスペースを作成します。 |
| **レイアウトの名前を変更…** |選択したワークスペースの名前を変更します。 |
| **レイアウトをリロード** |選択したワークスペースをすぐに適用します。 |
| **レイアウトを削除…** |選択したワークスペースとそのファイルを完全に削除します。 |

## 永続化設定

ワークスペース設定ページの下部では、Lumi が自動的に保存する内容を制御します。

- **終了時にウィンドウの位置を保存**: オンにすると、終了するたびにドックとウィンドウの位置がディスクに書き込まれます。
- **同じモニター上でウィンドウを開く**: 最後のセッション中に表示されていたモニター上の各ウィンドウを再度開きます。
- **終了時にツール オプションを保存**: 終了時に現在のツール設定を保存します。
- **終了時に入力デバイスの設定を保存**: 終了時にスタイラスとデバイスの設定を保存します。

これらの設定はワークスペースごとに適用され、各レイアウトは独自の保存された状態を独立して維持します。

## ワークフローの例

アーティストが複数のワークスペースを使用する方法は次のとおりです。

- **ペイント** — 大きなブラシ ドック、暖色系のパディング カラー ([環境設定] → [画像ウィンドウ] → [デフォルトの外観] で設定)、好みのテーマ バリアント
- **インク入力** — ガイドとキャンバス境界がオン、スクロールバーがオン ([環境設定] → [デフォルトの外観] で設定)、ニュートラル パディング カラー
- **ラフ** — ドックは非表示、ルーラーやグリッドなし、暗いパディング、キャンバススペースを最大化するためのコンパクトなアイコンサイズ
- **全画面フォーカス** - 全画面の外観とデフォルトの外観ではパディングの色と装飾設定が異なるため、全画面を切り替えるとまったく異なる作業環境が得られます
- **スクリプト** — スクリプト パネルが開き、読みやすくするためにフォント サイズが変更され、異なるアイコン セットが表示されます。
