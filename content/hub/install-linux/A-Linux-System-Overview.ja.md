Linux は、開発者の広大なコミュニティを持つ強力で多用途のオペレーティング システムです。 Linux システムの中核は、シームレスなユーザー エクスペリエンスを提供するために連携して動作するいくつかの主要なコンポーネントで構成されています。この概要では、カーネル、ディストリビューション、パッケージ マネージャー、ディスプレイ マネージャー、デスクトップ環境、ディスプレイ サーバー (X11 または Wayland) など、Linux システムの重要な部分の概要を説明します。

### 重要な用語

#### カーネル
ハードウェア (通常は Linux) と直接接続するオペレーティング システムの中核。

#### 配布
Linux ディストリビューション。カーネルとユーザー空間ツール、ライブラリ、およびソフトウェアがパッケージ化されています。例には、Debian、Arch Linux、Fedora などがあります。

#### パッケージマネージャー
ソフトウェア アプリケーションをリポジトリにインストール、更新、削除するために使用されるツール。例としては、Debian ベースのディストリビューションの APT、Fedora の DNF、Arch Linux の Pacman などがあります。

#### ディスプレイマネージャー
グラフィカルなログイン画面とセッションの開始を管理します。例には、GDM (GNOME ディスプレイ マネージャー)、LightDM、SDDM (シンプル デスクトップ ディスプレイ マネージャー) などがあります。

#### デスクトップ環境
グラフィカル ユーザー インターフェイス (GUI) を提供し、全体的な外観とユーザー エクスペリエンスを管理します。例としては、GNOME、Cinnamon、KDE ​​Plasma などがあります。

#### ディスプレイサーバー
ディスプレイの出力イベントと入力イベントを管理します。例としては、X11 (X Window System) や Wayland などがあります。 X11 は従来のディスプレイ サーバーですが、Wayland はより新しく、より安全な代替サーバーです。
