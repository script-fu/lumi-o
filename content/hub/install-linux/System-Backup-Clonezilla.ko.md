이전 버전으로 돌아가거나 손상된 데이터를 교체하기 위해 중요한 파일을 백업하는 것이 일반적입니다. 그러나 또 다른 필수 백업 유형은 시스템 상태의 전체 백업인 **디스크 복제**입니다.

시스템을 설정하고 제대로 작동하면 재해 발생 시 환경을 복원하기 위해 전체 백업을 만드는 것이 중요합니다. 이 백업은 작업 데이터를 정기적으로 저장하는 것을 보완합니다.

[Clonezilla](https://clonezilla.org/)은 무료 오픈 소스 디스크 이미징 및 복제 소프트웨어입니다. 이를 통해 사용자는 컴퓨터 하드 드라이브의 전체 백업을 생성하고 복원할 수 있으므로 IT 전문가와 가정 사용자 모두에게 널리 사용되는 도구입니다.

백업이 필요하지만 백업이 없는 것보다 백업이 있지만 필요하지 않은 것이 항상 더 좋습니다.


## Clonezilla의 주요 기능

- **디스크 이미징**: Clonezilla는 운영 체제, 애플리케이션, 데이터를 포함하여 하드 드라이브의 정확한 복사본을 생성합니다.
- **백업 및 복원**: 하드 드라이브의 백업 이미지를 생성하고 장애가 발생하거나 새 드라이브로 마이그레이션하는 경우 이를 복원할 수 있습니다.
- **무료 및 오픈 소스**: Clonezilla는 완전히 무료로 사용할 수 있으며 소스 코드를 수정하고 사용자 정의할 수 있습니다.


## Clonezilla를 사용하여 백업

### 준비 단계

Clonezilla용 USB 드라이브와 복제하려는 내부 드라이브보다 큰 외부 하드 드라이브가 필요합니다.

이러한 단계는 [official guide](https://clonezilla.org//fine-print-live-doc.php?path=./clonezilla-live/doc/01_Save_disk_image/00-boot-clonezilla-live-cd.doc#00-boot-clonezilla-live-cd.doc)을 기반으로 프로세스를 단순화합니다. 명확성을 높이기 위해 스크린샷이 포함된 전체 가이드를 검토하는 것이 좋습니다.

1. **Clonezilla Live USB 또는 CD/DVD 만들기**: Clonezilla [website](https://clonezilla.org/liveusb.php)의 자세한 지침에 따라 부팅 가능한 USB 또는 CD/DVD를 만듭니다.

2. **외부 백업 드라이브 연결**: 외부 드라이브를 연결하고 시스템에서 인식되는지 확인합니다. 이것이 백업 대상이 됩니다.

3. **파티션 레이아웃 확인**: 터미널에서 `lsblk` 명령을 사용하여 기본 하드 드라이브의 파티션 레이아웃을 확인합니다. 기본 장치 이름을 기록해 두십시오.

4. **Clonezilla Live USB 드라이브에서 부팅**: 컴퓨터를 다시 시작하고 생성한 Clonezilla 미디어에서 부팅합니다. BIOS/UEFI 설정에 액세스하고(일반적으로 부팅 중에 F2, F12, ESC 또는 DEL을 눌러) 부팅 순서를 조정하여 USB 드라이브의 우선 순위를 지정해야 할 수도 있습니다.



### Clonezilla를 사용한 백업

1. **백업 모드 선택**: Clonezilla가 부팅되면 "장치-장치" 모드를 선택합니다. 이 모드를 사용하면 내부 드라이브를 외부 장치에 직접 복제할 수 있습니다.

2. **소스 장치 선택**: 기본 내부 드라이브를 선택합니다.

3. **대상 장치 선택**: 외부 백업 드라이브를 대상 장치로 선택합니다. 중요한 데이터를 덮어쓰지 않도록 장치를 선택할 때 주의하세요. 대상 드라이브의 크기가 소스 드라이브보다 크거나 같은지 확인하십시오.

4. **백업 프로세스 시작**: Clonezilla가 백업 프로세스를 시작합니다. 파티션 크기와 드라이브 속도에 따라 이 작업은 몇 분에서 몇 시간까지 걸릴 수 있습니다.

5. **백업에 라벨을 붙입니다**: 백업이 완료된 후 USB 드라이브와 외장 하드 드라이브에 날짜와 백업한 시스템에 라벨을 붙입니다. 안전한 곳에 보관하세요.

---

### 백업에서 복원 중

백업에서 데비안 시스템을 복원해야 한다면 다음 단계를 따르세요:

1. **Clonezilla 미디어에서 부팅**: Clonezilla USB를 삽입하고 백업 프로세스와 동일한 단계에 따라 부팅합니다.2. **복원 모드 선택**: 다시 "장치-장치" 모드를 선택하지만 이번에는 백업 이미지에서 복원하게 됩니다. 이렇게 하면 외부 드라이브의 모든 데이터가 내부 드라이브로 다시 복사됩니다.

3. **소스 장치 선택**: 백업이 저장된 외부 드라이브를 선택합니다.

4. **대상 장치 선택**: 백업을 복원하려는 내부 드라이브를 선택합니다.

5. **복원 프로세스 시작**: Clonezilla가 복원 프로세스를 시작합니다. 백업과 마찬가지로 필요한 시간은 드라이브 크기와 하드웨어 속도에 따라 달라집니다.

## 최종 메모

Clonezilla를 사용한 디스크 백업은 운영 체제, 설정, 애플리케이션 등 전체 시스템이 보존되도록 보장합니다. 최소한의 노력으로 심각한 오류로부터 시스템을 보호하고 충돌 발생 시 가동 중지 시간을 최소화할 수 있습니다.

**백업은 필수적입니다**. 정기적으로 백업을 업데이트하고 주기적으로 테스트하여 필요할 때 시스템을 복원할 수 있는지 확인하십시오.

부팅한 후 외부 백업 드라이브를 연결하고 Linux의 디스크 유틸리티를 사용하여 파티션 구조를 검사할 수 있습니다. 백업 드라이브는 내부 드라이브의 구조를 미러링해야 하며 동일한 파티션이 있어야 하며 외부 드라이브가 더 큰 경우 일부 사용되지 않는 공간도 있어야 합니다.
