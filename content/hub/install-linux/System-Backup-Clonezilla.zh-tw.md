備份重要文件以返回早期版本或替換損壞的資料是很常見的。然而，另一種重要的備份類型是**磁碟克隆**，即係統狀態的完整備份。

一旦系統設定完畢並運作良好，建立完整備份對於在災難發生時復原環境至關重要。此備份是定期保存工作資料的補充。

[Clonezilla](https://clonezilla.org/) 是一款免費的開源磁碟映像和複製軟體。它允許用戶創建和恢復電腦硬碟的完整備份，使其成為 IT 專業人員和家庭用戶的流行工具。

有備份但不需要它總是比需要備份但沒有備份要好。


## Clonezilla 的主要特點

- **磁碟映像**：Clonezilla 建立硬碟的精確副本，包括作業系統、應用程式和資料。
- **備份和還原**：它使您能夠建立硬碟的備份映像並在發生故障或遷移到新磁碟機時還原它。
- **免費和開源**：Clonezilla 完全免費使用，原始碼可用於修改和自訂。


## 使用 Clonezilla 進行備份

### 準備步驟

您將需要一個用於 Clonezilla 的 USB 驅動器和一個比您要克隆的內部驅動器更大的外部硬碟。

這些步驟簡化了基於 [official guide](https://clonezilla.org//fine-print-live-doc.php?path=./clonezilla-live/doc/01_Save_disk_image/00-boot-clonezilla-live-cd.doc#00-boot-clonezilla-live-cd.doc) 的流程。最好查看完整的指南，其中包括螢幕截圖以提高清晰度。

1. **建立 Clonezilla Live USB 或 CD/DVD**：依照 Clonezilla [website](https://clonezilla.org/liveusb.php) 上的詳細說明建立可啟動 USB 或 CD/DVD。

2. **連接您的外部備份磁碟機**：插入外部磁碟機並確保您的系統可以識別它。這將是您備份的目的地。

3. **驗證您的分區佈局**：在終端機中使用 `lsblk` 命令驗證主硬碟的分區佈局。記下主要設備名稱。

4. **從 Clonezilla Live USB 驅動器啟動**：重新啟動電腦並從您建立的 Clonezilla 媒體啟動。您可能需要存取 BIOS/UEFI 設定（通常在啟動過程中按 F2、F12、ESC 或 DEL）並調整啟動順序以優先考慮 USB 驅動器。



### 使用 Clonezilla 備份

1. **選擇備援模式**：Clonezilla 啟動後，選擇「設備-設備」模式。此模式可讓您直接將內部磁碟機複製到外部裝置。

2. **選擇來源設備**：選擇主要內部驅動器。

3. **選擇目標裝置**：選擇您的外部備份磁碟機作為目標裝置。選擇設備時要小心，避免覆蓋重要數據。確保目標磁碟機的大小等於或大於來源磁碟機的大小。

4. **啟動備份程序**：Clonezilla 將啟動備份程序。根據分區的大小和驅動器的速度，這可能需要幾分鐘到幾個小時。

5. **標記您的備份**：備份完成後，在 USB 驅動器和外部硬碟上標記日期以及您備份的系統。將它們存放在安全的地方。

---

### 從備份還原

如果您需要從備份還原 Debian 系統，請依照下列步驟操作：

1. **從 Clonezilla 媒體啟動**：插入 Clonezilla USB 並從中啟動，並按照與備份過程中相同的步驟進行操作。2. **選擇復原模式**：再次選擇「裝置-裝置」模式，但這次，您將從備份映像中還原。這會將外部磁碟機中的所有資料複製回內部磁碟機。

3. **選擇來源裝置**：選擇儲存備份的外接磁碟機。

4. **選擇目標裝置**：選擇要還原備份的內部磁碟機。

5. **開始恢復過程**：Clonezilla 將開始恢復過程。與備份一樣，所需的時間取決於磁碟機的大小和硬體的速度。

## 最後的註釋

使用 Clonezilla 進行磁碟備份可確保保留您的整個系統（作業系統、設定和應用程式）。只需付出最少的努力，您就可以保護您的系統免受災難性故障的影響，並最大限度地減少崩潰時的停機時間。

請記住，**備份至關重要**。定期更新您的備份並定期測試它們，以確保您可以在需要時還原系統。

啟動後，您可以插入外部備份磁碟機並使用 Linux 中的磁碟公用程式檢查其分割區結構。備份磁碟機應鏡像內部磁碟機的結構，具有相同的分割區和一些未使用的空間（如果外部磁碟機較大）。
