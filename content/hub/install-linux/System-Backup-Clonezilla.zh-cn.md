备份重要文件以返回到早期版本或替换损坏的数据是很常见的。然而，另一种重要的备份类型是**磁盘克隆**，即系统状态的完整备份。

一旦系统设置完毕并运行良好，创建完整备份对于在发生灾难时恢复环境至关重要。此备份是对定期保存工作数据的补充。

[Clonezilla](https://clonezilla.org/) 是一款免费的开源磁盘映像和克隆软件。它允许用户创建和恢复计算机硬盘的完整备份，使其成为 IT 专业人员和家庭用户的流行工具。

有备份但不需要它总是比需要备份但没有备份要好。


## Clonezilla 的主要特点

- **磁盘映像**：Clonezilla 创建硬盘驱动器的精确副本，包括操作系统、应用程序和数据。
- **备份和恢复**：它使您能够创建硬盘驱动器的备份映像并在发生故障或迁移到新驱动器时恢复它。
- **免费和开源**：Clonezilla 完全免费使用，源代码可用于修改和定制。


## 使用 Clonezilla 进行备份

### 准备步骤

您将需要一个用于 Clonezilla 的 USB 驱动器和一个比您要克隆的内部驱动器更大的外部硬盘驱动器。

这些步骤简化了基于 [official guide](https://clonezilla.org//fine-print-live-doc.php?path=./clonezilla-live/doc/01_Save_disk_image/00-boot-clonezilla-live-cd.doc#00-boot-clonezilla-live-cd.doc) 的流程。最好查看完整的指南，其中包括屏幕截图以提高清晰度。

1. **创建 Clonezilla Live USB 或 CD/DVD**：按照 Clonezilla [website](https://clonezilla.org/liveusb.php) 上的详细说明创建可启动 USB 或 CD/DVD。

2. **连接您的外部备份驱动器**：插入外部驱动器并确保您的系统可以识别它。这将是您备份的目的地。

3. **验证您的分区布局**：在终端中使用 `lsblk` 命令验证主硬盘驱动器的分区布局。记下主要设备名称。

4. **从 Clonezilla Live USB 驱动器启动**：重新启动计算机并从您创建的 Clonezilla 介质启动。您可能需要访问 BIOS/UEFI 设置（通常在启动过程中按 F2、F12、ESC 或 DEL）并调整启动顺序以优先考虑 USB 驱动器。



### 使用 Clonezilla 备份

1. **选择备份模式**：Clonezilla 启动后，选择“设备-设备”模式。此模式允许您直接将内部驱动器克隆到外部设备。

2. **选择源设备**：选择主要内部驱动器。

3. **选择目标设备**：选择您的外部备份驱动器作为目标设备。选择设备时要小心，避免覆盖重要数据。确保目标驱动器的大小等于或大于源驱动器的大小。

4. **启动备份过程**：Clonezilla 将启动备份过程。根据分区的大小和驱动器的速度，这可能需要几分钟到几个小时。

5. **标记您的备份**：备份完成后，在 USB 驱动器和外部硬盘驱动器上标记日期以及您备份的系统。将它们存放在安全的地方。

---

### 从备份恢复

如果您需要从备份恢复 Debian 系统，请按照以下步骤操作：

1. **从 Clonezilla 介质启动**：插入 Clonezilla USB 并从中启动，按照与备份过程中相同的步骤进行操作。2. **选择恢复模式**：再次选择“设备-设备”模式，但这一次，您将从备份映像中恢复。这会将外部驱动器中的所有数据复制回内部驱动器。

3. **选择源设备**：选择存储备份的外部驱动器。

4. **选择目标设备**：选择要恢复备份的内部驱动器。

5. **开始恢复过程**：Clonezilla 将开始恢复过程。与备份一样，所需的时间取决于驱动器的大小和硬件的速度。

---

## 最后的注释

使用 Clonezilla 进行磁盘备份可确保保留您的整个系统（操作系统、设置和应用程序）。只需付出最少的努力，您就可以保护您的系统免受灾难性故障的影响，并最大限度地减少崩溃时的停机时间。

请记住，**备份至关重要**。定期更新您的备份并定期测试它们，以确保您可以在需要时恢复系统。

启动后，您可以插入外部备份驱动器并使用 Linux 中的磁盘实用程序检查其分区结构。备份驱动器应镜像内部驱动器的结构，具有相同的分区和一些未使用的空间（如果外部驱动器更大）。
